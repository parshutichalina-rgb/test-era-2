# ERA2 Queue Feature

Реализация экрана `Очередь генераций` и глобального статус-бара генераций на клиентских моках (без backend).

## Запуск

- `yarn dev` — локальная разработка
- `yarn build` — production build
- `yarn preview` — preview build
- `yarn lint` — линтер

## Роутинг

Проект использует кастомный роутер на History API (`src/shared/routing`).

- Страница очереди доступна по пути `"/queue"`.
- Переходы выполняются через `useNavigate`/`Link`.
- Глобальный статус-бар монтируется на уровне `App` и ведет на `"/queue"` из любого экрана.

## Персистентность очереди

Состояние очереди сохраняется в `localStorage` под ключом `era2_generation_queue_state_v1`.

При восстановлении после перезагрузки:

- задачи в статусе `running` переводятся в `queued`,
- остальные статусы восстанавливаются как есть.

Такой вариант выбран, чтобы не продолжать "висящие" интервалы и не показывать ложный прогресс после hard reload.

## Шрифты

По умолчанию используются `Geist Variable` и `Geist Mono Variable` (подключены в глобальных стилях).
Если Geist недоступен, фолбэк для sans — `Inter`, `Manrope`, `system-ui`; для mono — `JetBrains Mono`, `ui-monospace`.
